# MVP 任务

## MVP 目标

构建“仿真顾客决策平台”的第一版可点击原型。

第一版不需要真实后端和真实算法。

第一版重点是验证：

* 平台型 Agent 首页
* 左侧资产与工作区导航
* 中间任务启动、快捷功能和模板
* 首页到任务详情页的数据贯通
* 对话内容详情页 / 任务详情页
* 决策流程
* 仿真顾客表达
* 多视角反馈
* 风险识别
* 决策建议报告

## 技术要求

请根据当前项目技术栈实现。

如果项目还没有技术栈，优先使用：

* React
* TypeScript
* Vite
* Tailwind CSS

如果已有项目结构，请遵循已有结构，不要强行重建。

## MVP 页面

第一版包括六个页面：

1. 首页 / Agent 工作台
2. 对话内容详情页 / 任务详情页
3. 决策输入页
4. 仿真顾客群体页
5. 决策推演与反馈页
6. 决策建议报告页

任务：

* 创建基础目录结构
* 创建 mock data
* 创建页面路由
* 创建通用 Layout

验收：

https://enter.converge.ai/workspace/8fe43b52-efb8-40b8-83a5-59aa0d20560d?tab=newchat

若无法访问该链接，应要求用户提供截图，或仅按本文档描述进行保守调整。

* 可以优化左侧导航，但核心入口必须可发现。
* 右侧顾客正在想必须保留，这是本产品核心差异。
* 中间输入框必须保持任务启动器心智。
* 不要把首页做成通用聊天机器人首页。

* 设置弹窗默认展示“关于你”表单。
* 表单字段包含：昵称、职业、你的详情、记忆开关。

中间必须展示：

首页主文案建议：

今天要验证什么企业决策？

输入框占位文案建议：

输入一个决策，例如：会员年费是否从 199 元提高到 299 元？

注意：

这个输入框是任务启动器，不是通用聊天窗口。

### 首页交互要求

用户在首页输入框输入内容并发送后，必须进入对话内容详情页。

必须传递：

* 用户输入内容
* 选择的能力按钮，默认可为“调研问卷”
* 选择的业务场景
* 当前引用的资源和知识

禁止在详情页展示与首页输入不一致的写死第一段对话。

## 阶段 3：对话内容详情页 / 任务详情页

页面路径建议：

/task/:taskId

#task

组件建议：

* CollapsedSidebar
* TaskHeader
* ConversationTimeline
* OriginalRequestBlock
* ClarificationQuestionCard
* SurveyPlanCard
* SimulationProgress
* SurveyConclusionReport
* CustomerVoicePanel
* CustomerVoiceCard
* CustomerQuickProfilePopover
* TaskInputBar

布局要求：

* 左侧参考 AnyGen 任务页收起导航栏
* 中间展示任务对话与结构化工作流
* 右侧展示顾客留声区

默认任务类型：

调研问卷

正常流程：

1. 展示首页传入的用户原始输入
2. Agent 提出 2 个澄清问题
3. 用户勾选问题选项
4. 生成调研方案
5. 开始让仿真顾客回答
6. 展示回答进度
7. 顾客全部回复后展示调研结论

### 当前确认 UI 细节

中间区域：

* 用户原始输入必须显示用户头像和消息气泡。
* Agent 澄清问题必须显示机器人头像和可折叠卡片。
* Agent 调研方案必须显示机器人头像和可折叠卡片。
* 澄清问题和调研方案右上角需要有展开 / 收起按钮。
* checkbox 选项要像问卷选项，不要做成强营销感标签。
* 点击“生成调研方案”后不自动收起澄清问题卡，折叠由用户手动控制。

右侧区域：

* 顾客留声卡片 hover 时显示快速档案。
* hover 只允许出现一层弹窗，禁止双层弹窗。
* 快速档案展示脱敏信息、顾客留言摘要、已购商品和仿真顾客人设。
* 快速档案不展示支持 / 反对 / 犹豫 / 条件接受标签。

澄清问题 1：

你更想验证哪一类顾客？

选项：

* 当前会员
* 潜在会员
* 高价值用户
* 价格敏感用户

澄清问题 2：

你更关心哪一个决策风险？

选项：

* 涨价接受度
* 老用户流失
* 权益感知
* 竞品比较

调研方案必须展示：

* 调研目标
* 目标顾客
* 核心假设
* 问卷模块
* 关键问题
* 预计输出

右侧顾客留声必须展示：

* 头像
* 脱敏昵称
* 年龄
* 门店 / 渠道
* 会员身份
* 本次任务相关回答
* 回答状态

右侧不要展示：

* 输入框
* 筛选器
* 价格敏感型等分析标签
* 支持 / 反对等态度标签

验收：

* 页面结构与确认的 UI 线框图 1:1 对齐。
* 首页输入文本与详情页第一段用户原始输入一致。
* 点击“生成调研方案”后才出现调研方案。
* 点击“开始让仿真顾客回答”后才进入回答进度。
* 顾客回答完成后才出现调研结论。
* 右侧顾客留声围绕当前任务内容，不使用无关固定文案。
* 右侧顾客 hover 快速档案只出现一层。
* 澄清问题卡和调研方案卡可以正常收起和展开。

## UI 要求

* 中文 UI 文案
* 首页借鉴平台型 Agent 产品：左侧资产导航，右侧任务启动、快捷功能和模板
* 任务详情页借鉴 AnyGen 任务页：左侧收起导航，中间任务工作流，右侧顾客留声
* 已确认的 UI 线框图是实现标准，不能只实现大概布局
* 不要做成普通聊天机器人界面
* 不要一打开就是复杂后台表单
* 任务内页面像企业决策驾驶舱
* 信息层级清楚
* 支持、反对、犹豫、条件接受要明显区分
* 顾客原声要突出
* 风险和下一步建议要清楚

## 验收标准

MVP 完成后，应满足：

1. 六个页面都可以访问。
2. 页面之间可以跳转。
3. 首页能表达平台型 Agent 的信息架构。
4. 左侧能看到 Skill 库、Tool 库、资源库、知识库、项目库、历史记录和我的。
5. 首页中间能看到任务启动器、能力按钮、场景按钮和模板。
6. 首页右侧能看到顾客正在想。
7. 首页输入内容必须进入任务详情页，并成为第一段用户原始输入。
8. 任务详情页必须能完成调研问卷正常流程。
9. 数据虽然是 mock，但结构清晰。
10. 用户能看到仿真顾客如何影响决策。
11. 用户能看到支持、反对、犹豫和条件接受。
12. 用户能看到风险和下一步行动建议。
13. 产品看起来像平台型 Agent，不像普通聊天机器人。
14. 代码结构清晰，方便后续接入真实 Agent 服务。

## 下一步开发指令建议

先不要直接进入四个决策流程页。

第一轮开发应先实现首页 / Agent 工作台，确认平台型 Agent 的信息架构和首页气质正确，再进入任务内的决策输入页、仿真顾客页、反馈页和报告页。
