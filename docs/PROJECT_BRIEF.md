# 项目简报

## 项目名称

暂定名称：

仿真顾客决策平台
Customer Simulation Decision Platform

## 一句话定位

一个面向企业的仿真顾客决策平台型 Agent 工作台，帮助企业在做产品、营销、定价、品牌、渠道、服务等关键决策前，调度 Skill、Tool、资源库、知识库和仿真顾客群体，模拟目标顾客反应，识别风险，并生成可执行的决策建议。

## 产品要解决的问题

企业在做重要决策时，经常面临以下问题：

1. 决策者过度依赖内部视角。
2. 团队以为顾客会理解，但顾客可能并不理解。
3. 团队以为顾客会买单，但顾客可能有强烈顾虑。
4. 决策前缺少足够多样化的顾客反馈。
5. 真实调研周期长、成本高、样本组织困难。
6. 决策风险往往在执行后才暴露。
7. 用户研究、市场分析、业务决策之间没有形成闭环。
8. 企业不知道哪些顾客支持、哪些顾客反对、哪些顾客需要被进一步说服。

本产品要解决的核心问题是：

帮助企业在正式执行决策前，提前模拟目标顾客的认知、情绪、动机、疑虑和行动倾向。

## 平台型 Agent 首页

产品首页不是普通聊天机器人，也不是复杂后台表单。

首页应采用平台型 Agent 工作台结构，可综合参考 AnyGen、Converge AI、ChatGPT 和 Lovart 的布局方式。

推荐首页分为三个区域：

左侧是平台资产与工作区：

* 新建任务
* 项目库
* Skill 库
* Tool 库
* 资源库
* 知识库
* 历史记录
* 我的

左下角“我的”参考 ChatGPT 的账户入口：

* 点击后弹出账户菜单
* 菜单包含升级套餐、个性化、个人资料、设置、帮助和退出登录
* 点击“个性化”打开设置弹窗
* 设置弹窗默认展示“关于你”表单，用于填写昵称、职业、个人详情和记忆开关

该能力属于平台工作台基础体验，不改变核心主线；核心主线仍然是“仿真顾客帮助企业做决策”。

中间是任务启动区：

* 直接对话框：输入“今天要验证什么企业决策？”
* 快捷功能：辅助设计、统计分析、调研问卷、群体决策、爆款图谱
* 模板：定价决策、新产品决策、营销文案、品牌定位、会员权益、服务政策、渠道策略、功能优先级

右侧是仿真顾客心声预览：

* 顾客正在想
* 脱敏顾客信息
* 真实消费心理活动

其中，直接对话框是任务启动器，不是闲聊入口。

## 资源库与知识库

资源库是 Agent 的原材料，主要放结构化业务资产和素材资产，例如商品表、SKU 表、价格表、会员表、仿真顾客表、竞品表、渠道表、营销素材和设计素材。

知识库是 Agent 的背景理解，主要放企业知识、规则和长期上下文，例如品牌定位、用户研究报告、产品策略、会员体系、服务政策、价格策略原则、历史决策复盘和企业 SOP。

## 核心价值

本产品帮助企业回答：

1. 这个决策顾客会怎么理解？
2. 哪些顾客会支持？
3. 哪些顾客会反对？
4. 哪些顾客会犹豫？
5. 顾客最大的顾虑是什么？
6. 顾客被打动的理由是什么？
7. 当前方案需要如何修改？
8. 这个决策是应该推进、调整、小范围测试，还是暂缓？
9. 下一步行动应该是什么？

## 产品哲学

本产品不是为了替代真实顾客研究。

它的价值是：

在企业正式投入资源之前，提供一个低成本、高效率、多视角的“顾客预演场”。

它帮助企业更早发现问题，而不是等产品上线、广告投放、价格调整或服务政策变更之后才发现顾客不买单。

## 核心流程

1. 在首页输入企业决策问题或选择模板
2. 选择或补充项目、资源和知识上下文
3. 拆解决策影响因素
4. 生成仿真顾客群体
5. 收集不同顾客反馈
6. 汇总支持、反对、犹豫和条件接受
7. 识别关键风险
8. 生成方案优化建议
9. 输出决策报告
10. 记录决策并支持后续复盘

## 深度访谈支线

除主任务流程外，首页右侧顾客心声可以进入深度访谈。

该支线用于把单个顾客的心理活动展开为决策证据：

1. 用户点击首页某位仿真顾客的一句话
2. 进入一对一深访页
3. 顾客原话成为第一条访谈消息
4. 用户继续追问顾客动机、顾虑和接受条件
5. 系统提示是否进入顾客圆桌模式
6. 圆桌模式中引入用户、真人成员、更多仿真顾客和专家 Agent
7. 输出当前发言者档案、访谈结论草稿和下一步追问方向

深度访谈不是普通私聊，而是对“仿真顾客证据”的进一步挖掘。
