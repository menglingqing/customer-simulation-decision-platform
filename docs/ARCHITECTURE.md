# 架构设计

## 架构总览

本产品采用“平台型 Agent 工作台 + 垂直决策推演 Agent”的架构。

整体架构分为九层：

1. 平台首页与任务启动层
2. 工作区与项目层
3. Skill 库层
4. Tool 库层
5. 资源库层
6. 知识库层
7. 决策主控 Agent 层
8. 仿真顾客与专业子 Agent 层
9. 决策证据、报告与复盘层

## 1. 平台首页与任务启动层

首页是平台型 Agent 的入口。

用户可以通过三种方式启动任务：

* 在直接对话框输入要验证的企业决策
* 点击快捷功能，例如生成仿真顾客、对比两个方案、识别反对理由
* 选择业务模板，例如定价决策、新产品决策、营销文案、品牌定位

首页不应设计成普通聊天机器人，也不应一开始就是复杂表单。

当前静态原型约定：

* 首页输入框负责创建任务上下文。
* 当前任务上下文暂存在前端本地状态 / localStorage 中。
* 用户按回车发送后进入任务详情页。
* 任务详情页第一段用户输入必须读取任务上下文，不允许写死示例。

## 1.1 任务详情页交互层

任务详情页是首页任务启动后的第一承接页面。

当前默认流程为“调研问卷”：

1. 展示用户原始输入
2. Agent 提出两个澄清问题
3. 用户勾选关注顾客和决策风险
4. Agent 生成调研方案
5. 用户启动仿真顾客回答
6. 右侧顾客留声展示回答状态
7. 回答完成后中间展示调研结论

中间区域采用“消息头像 + 结构化工作流卡片”：

* 用户消息使用用户头像。
* Agent 卡片使用机器人头像。
* 澄清问题卡和调研方案卡支持折叠 / 展开。
* 折叠状态属于 UI 状态，不影响任务数据。

右侧顾客留声采用“卡片列表 + 单层 hover 快速档案”：

* 顾客卡片展示轻量回答。
* hover 头像或卡片展示快速档案。
* 快速档案只保留一套弹窗机制，避免双层弹窗。
* 快速档案展示脱敏信息、已购商品和仿真人设，不展示分析标签。

## 2. 工作区与项目层

项目库用于承载一个个企业决策项目。

一个项目可以包含：

* 决策问题
* 引用的资源和知识
* 选择的 Skill 和 Tool
* 仿真顾客群体
* 推演反馈
* 决策报告
* 复盘记录

## 3. Skill 库层

Skill 库表示 Agent 能做什么。

典型 Skill 包括：

* 决策拆解
* 仿真顾客生成
* 顾客反馈模拟
* 争议识别
* 风险评估
* 方案优化
* 报告生成
* 决策复盘

## 4. Tool 库层

Tool 库表示 Agent 能调用什么外部或内部工具。

典型 Tool 包括：

* 表格读取
* 数据分析
* 文档解析
* 问卷生成
* 报告导出
* 图表生成
* 文件上传
* 搜索与检索

## 5. 资源库层

资源库是 Agent 的原材料，主要放结构化业务资产和素材资产。

示例：

* 商品表、SKU 表、价格表
* 用户表、会员表、仿真顾客表
* 竞品表、渠道表、门店表
* 营销素材、设计素材、历史活动素材
* 问卷结果表、访谈记录整理表

## 6. 知识库层

知识库是 Agent 的背景理解，主要放企业知识、规则和长期上下文。

示例：

* 品牌定位
* 用户研究报告
* 产品策略文档
* 会员体系说明
* 服务政策
* 价格策略原则
* 历史决策复盘
* 行业洞察
* 企业内部 SOP

## 7. 决策主控 Agent 层

决策主控 Agent 负责组织整个推演过程。

职责包括：

* 理解决策问题
* 判断决策类型
* 选择需要调用的 Skill、Tool、资源和知识
* 拆解决策变量
* 选择需要生成的仿真顾客类型
* 调度子 Agent
* 汇总反馈
* 形成最终建议

## 8. 仿真顾客与专业子 Agent 层

建议包括：

1. 决策拆解 Agent
2. 仿真顾客生成 Agent
3. 顾客反馈模拟 Agent
4. 争议识别 Agent
5. 风险评估 Agent
6. 方案优化 Agent
7. 决策报告 Agent
8. 决策复盘 Agent

仿真顾客群体应代表不同的顾客认知、动机、偏好、预算、场景、信任程度和决策方式。

## 9. 决策证据、报告与复盘层

系统输出的报告应包括：

* 总体建议
* 支持理由
* 反对理由
* 犹豫原因
* 关键风险
* 机会点
* 方案修改建议
* 需要验证的问题
* 下一步行动建议

系统还应记录：

* 企业做过哪些决策
* 当时调用了哪些资源和知识
* 当时仿真顾客如何反馈
* 企业最终选择了什么方案
* 执行后的真实结果如何
* 哪些预测准确
* 哪些判断偏差
* 下次决策应如何调整
