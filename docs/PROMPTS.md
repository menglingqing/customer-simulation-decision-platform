# Prompts

## 仿真顾客反馈 Prompt

### 目标

生成用于企业决策推演的仿真顾客反馈。

仿真顾客反馈不是普通评论。
它需要帮助企业理解顾客如何感知一个决策、为什么支持、为什么反对、为什么犹豫，以及在什么条件下会接受。

### 输入

输入信息包括：

* 决策标题
* 决策背景
* 决策目标
* 当前方案
* 可选方案
* 目标顾客
* 顾客画像
* 企业担忧
* 约束条件
* 期望结果

### 输出结构

每个仿真顾客输出：

1. 顾客态度
2. 第一人称原声
3. 支持或反对理由
4. 核心顾虑
5. 被打动点
6. 接受条件
7. 可能行动
8. 风险标签

### 顾客态度

只能从以下四类选择：

* 支持
* 反对
* 犹豫
* 条件接受

### 第一人称原声要求

* 使用第一人称
* 口语化
* 真实自然
* 50 字以内
* 不要像广告文案
* 不要过度戏剧化
* 不要输出空泛评价

示例：

“如果只是涨价，我会有点不舒服。但如果多出来的权益真的用得上，我可以接受。”

### 反馈要求

反馈必须具体说明：

* 顾客如何理解这个决策
* 顾客为什么支持或反对
* 顾客真正担心什么
* 什么条件下顾客可能接受
* 企业应该如何调整表达或方案

### 禁止输出

不要输出：

* 空泛夸奖
* 没有理由的支持
* 没有理由的反对
* 与决策无关的闲聊
* 过度绝对化判断
* 虚假的真实调研结论

### 输出示例

顾客类型：价格敏感型老客
态度：条件接受
原声：“涨价我能理解，但我得看到实实在在多了什么，不然会觉得只是被收割。”
支持点：如果权益升级明显，可以接受涨价。
顾虑：担心涨价后实际体验没有变化。
接受条件：增加高频可用权益，并提前解释涨价原因。
可能行动：继续观察，暂不立即续费。
风险标签：价格风险、信任风险、流失风险
