# Agent 设计

## 总体设计

本系统采用多 Agent 协作架构。

用户在前台看到的是业务决策流程。
后台由决策主控 Agent 调度多个专业子 Agent，完成决策拆解、顾客模拟、反馈推演、风险识别和建议生成。

## 1. 决策主控 Agent

职责：

* 理解企业输入的决策问题
* 判断决策类型
* 规划推演流程
* 调用相关子 Agent
* 汇总结果
* 生成最终报告

## 2. 决策拆解 Agent

职责：

* 拆解决策目标
* 拆解决策影响对象
* 拆解决策变量
* 识别可能风险
* 判断需要哪些顾客视角

示例：

“会员年费从 199 元涨到 299 元”可以拆解为：

* 价格变化
* 权益感知
* 老用户流失风险
* 新用户转化阻力
* 品牌信任问题
* 竞品比较问题

## 3. 仿真顾客生成 Agent

职责：

* 根据决策场景生成顾客群体
* 保证顾客类型多样
* 避免顾客人设单一
* 生成对决策有判断价值的顾客差异

## 4. 顾客反馈模拟 Agent

职责：

* 模拟每类顾客的理解方式
* 模拟支持理由
* 模拟反对理由
* 模拟犹豫点
* 模拟转化条件
* 输出第一人称顾客原声

## 5. 争议识别 Agent

职责：

* 识别顾客反馈中的冲突点
* 总结主要争议
* 判断哪些争议可以通过沟通解决
* 判断哪些争议属于产品或策略本身的问题

## 6. 风险评估 Agent

职责：

* 识别流失风险
* 识别转化风险
* 识别品牌信任风险
* 识别价格敏感风险
* 识别传播风险
* 识别执行风险

## 7. 方案优化 Agent

职责：

* 根据顾客反馈修改方案
* 提出替代方案
* 提出沟通策略
* 提出权益补强建议
* 提出小范围测试建议

## 8. 决策报告 Agent

职责：

* 汇总推演结果
* 输出决策建议
* 输出行动清单
* 输出风险说明
* 输出置信度
* 生成适合企业用户阅读的报告

## 9. 决策复盘 Agent

职责：

* 记录历史决策
* 记录仿真预测
* 记录真实执行结果
* 对比预测和结果
* 更新后续决策建议
