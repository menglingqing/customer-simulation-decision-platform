# 页面规格

## MVP 页面总览

第一版 MVP 包括八个核心页面：

1. 首页 / Agent 工作台
2. 对话内容详情页 / 任务详情页
3. 深度访谈页 / 顾客圆桌模式
4. 探索技能详情页
5. 决策输入页
6. 仿真顾客群体页
7. 决策推演与反馈页
8. 决策建议报告页

页面之间形成完整闭环：

首页启动任务
→ 创建任务上下文
→ 进入任务详情页澄清意图
→ 生成调研方案或决策方案
→ 生成仿真顾客
→ 推演反馈
→ 输出建议报告

## 页面 1：首页 / Agent 工作台

### 页面目标

让用户像使用平台型 Agent 一样启动企业决策仿真任务，并在首页就感受到“仿真顾客在场”。

### UI 参考优先级

首页 UI 可参考以下产品，但需要按本项目主线取舍：

1. AnyGen：参考平台型 Agent 工作台、左侧导航、中间任务启动器和模板区。
2. Converge AI：参考 workspace 页面布局、新建对话 / 新建任务入口、左侧历史记录与中间输入区的空间组织。
3. ChatGPT：参考最近记录、项目和基础导航的简洁侧栏。
4. Lovart：参考大输入框与胶囊能力按钮的低噪音表达。

Converge AI 参考链接：

https://enter.converge.ai/workspace/8fe43b52-efb8-40b8-83a5-59aa0d20560d?tab=newchat

注意：

该链接可能需要登录态或 workspace 权限。若 Claude Code 或其他开发工具无法访问，应以用户提供的截图为准，而不是凭空猜测细节。

### 页面核心问题

用户在这里要回答：

今天要验证什么企业决策？
这个任务需要使用哪些能力、资源或知识？
我要从模板开始，还是直接输入任务？
是否继续一个历史项目？

### 左侧导航

左侧参考 AnyGen 的轻导航结构，应包含：

1. 顾客搭档
2. 新建
3. 探索技能
4. 资源库
5. 知识库
6. 顾客群体库
7. 项目 / 新建项目
8. 最近项目
9. 我的

不要在一级导航放“我的 Persona”。“我的”只代表当前登录用户、账号和企业空间。单个仿真顾客画像从“顾客群体库”内部进入。

左侧导航设计补充：

* 可以参考 Converge AI 的 workspace 侧栏处理方式，强化工作区、历史记录和新建任务入口。
* 左侧可以做减法，但不能牺牲可读性和可用性。
* 如果使用图标入口，必须保留清晰 hover tooltip 或可展开文本。
* “我的”必须在常规视口高度内可见。
* 不应为了极简而把核心入口隐藏到难以发现。

### 左下角账户菜单

首页左下角“我的”区域参考 ChatGPT 的账户入口设计。

交互要求：

* 点击“我的 / 企业空间”区域后，在左下角上方弹出账户菜单。
* 菜单弹出时，底部账号入口仍保持可见。
* 菜单不应遮挡首页中间任务启动器。
* 再次点击账号入口可以关闭菜单。

菜单项建议：

* 升级套餐
* 个性化
* 个人资料
* 设置
* 帮助
* 退出登录

### 个性化设置弹窗

点击账户菜单中的“个性化”后，打开设置弹窗，并默认定位到“关于你”表单。

弹窗结构参考 ChatGPT 设置页：

* 左侧：设置导航
* 右侧：当前设置内容

左侧导航建议：

* 常规
* 通知
* 个性化
* 应用
* 账户

“关于你”表单字段：

* 昵称
* 职业
* 你的详情
* 记忆开关

说明：

* 当前 MVP 只需要实现前端静态交互，不需要真实保存。
* “你的详情”用于表达用户希望平台记住的行业背景、决策偏好、常用业务场景或沟通风格。
* 记忆开关用于模拟未来个性化上下文能力，不影响当前 mock 数据。

### 中间主区域

中间参考 AnyGen 的首页结构，应包含：

1. 直接对话框 / 任务启动器
2. Lovart 式胶囊能力按钮
3. 业务场景按钮
4. 模板区

中间主区域设计补充：

* 可以参考 Converge AI 的新建对话页，让用户一眼知道从哪里开始任务。
* 输入区应保持视觉中心地位，周围能力入口只作为辅助。
* 首页不是复杂后台，不应堆叠过多说明、状态、统计和装饰。
* 若增加 workspace 级信息，必须服务于“发起企业决策仿真任务”。

### 直接对话框

标题建议：

今天要验证什么企业决策？

占位文案：

输入一个决策，例如：会员年费是否从 199 元提高到 299 元？

注意：

这个输入框是任务启动器，不是通用聊天窗口。

用户在输入框内输入内容并发送后，必须进入“对话内容详情页 / 任务详情页”。首页输入内容必须作为任务详情页的第一段用户原始输入展示，不能在详情页使用写死的示例文案覆盖。

### 能力按钮

输入框下方第一行保留五个能力：

* 辅助设计
* 统计分析
* 调研问卷
* 群体决策
* 爆款图谱

不要在首页能力按钮中放“生成决策报告”“复盘历史决策”“更多”。

### 业务场景按钮

第二行展示业务场景：

* 定价决策
* 产品决策
* 营销增长
* 品牌定位
* 会员权益
* 服务政策

### 模板

建议包括：

* 定价决策
* 新产品决策
* 营销文案决策
* 品牌定位决策
* 会员权益决策
* 渠道策略决策
* 服务政策决策
* 功能优先级决策

### 资源库与知识库入口说明

资源库展示 Agent 可使用的业务原材料，例如商品表、SKU 表、会员表、仿真顾客表、价格表、竞品表、营销素材和设计素材。

知识库展示 Agent 可引用的企业背景理解，例如品牌定位、用户研究报告、产品策略、会员体系、服务政策、历史复盘和企业 SOP。

### 右侧顾客心声区

右侧只展示“顾客正在想”，不放输入框、筛选器或分析标签。

每条顾客心声展示：

* 头像
* 脱敏昵称
* 年龄
* 门店 / 渠道
* 会员身份
* 一句真实消费心理活动

示例：

符*儿，18 岁，杭州砂之船折扣店，女王卡：

“我会先看这个品牌是不是适合我，便宜不一定就会买。”

## 页面 2：对话内容详情页 / 任务详情页

### 页面目标

承接用户从首页发起的任务，并以“对话驱动的结构化工作流”推进任务完成。

默认任务类型为“调研问卷”。用户从首页输入内容后，页面需要先澄清用户意图，再生成调研方案，随后让仿真顾客回答，最后在中间区域输出调研结论。

### 页面布局

页面保持左中右三栏：

1. 左侧：AnyGen 式收起导航栏
2. 中间：任务对话与结构化工作流
3. 右侧：顾客留声区

### 左侧收起导航

左侧参考 AnyGen 任务页的收起状态，只保留图标入口。

建议包含：

* Logo / 首页
* 顾客搭档
* 新建
* 探索技能
* 搜索
* 资源库
* 知识库
* 项目库
* 历史记录
* 我的
* 通知
* 设置

左侧在任务页的职责是快速切换能力和资产，不抢主流程注意力。

### 中间任务工作流

中间区域必须展示完整任务推进过程，而不是普通聊天消息堆叠。

默认“调研问卷”正常流程包括：

1. 用户原始输入
2. Agent 澄清意图
3. 用户勾选澄清问题
4. 生成调研方案
5. 开始让仿真顾客回答
6. 展示顾客回答进度
7. 顾客全部回复完成后展示调研结论

### 中间区域 UI 结构

当前确认版本采用“消息头像 + 结构化工作流卡片”的表现：

* 用户原始输入：左侧展示用户头像，右侧展示轻量消息气泡。
* Agent 澄清问题：左侧展示机器人头像，右侧展示可折叠工作流卡片。
* 调研方案：左侧展示机器人头像，右侧展示可折叠工作流卡片。
* 顾客回答进度和调研结论：作为任务阶段卡片展示，避免做成闲聊回复。

可折叠卡片规则：

* 右上角使用展开 / 收起图标。
* 展开状态展示完整内容。
* 收起状态只保留标题与关键摘要。
* 点击“生成调研方案”后不强制收起澄清问题卡，是否收起由用户决定。
* 折叠状态只影响当前前端展示，不影响任务数据。

### 用户原始输入

第一段内容必须来自首页输入框。

示例：

用户在首页输入：

“我想了解会员涨价后，顾客是否还能接受 299 元年费。”

任务详情页第一段必须展示同一内容，不允许写死为其他示例。

### 澄清问题

默认“调研问卷”任务先提出 2 个聚焦问题。

问题 1：

你更想验证哪一类顾客？

选项：

* 当前会员
* 潜在会员
* 高价值用户
* 价格敏感用户

问题 2：

你更关心哪一个决策风险？

选项：

* 涨价接受度
* 老用户流失
* 权益感知
* 竞品比较

用户通过 checkbox 式选项完成，不要求输入长文本。选项视觉应接近调研问卷，而不是营销标签或大号胶囊按钮。

### 调研方案

用户确认后生成结构化调研方案。

调研方案至少包含：

* 调研目标
* 目标顾客
* 核心假设
* 问卷模块
* 关键问题
* 预计输出

示例问卷模块：

1. 当前会员价值感知
2. 涨价接受区间
3. 权益补强偏好
4. 流失风险判断
5. 沟通方式偏好

### 顾客回答进度

用户点击“开始让仿真顾客回答”后，中间区域展示收集进度。

进度展示包括：

* 已回答数量 / 总顾客数量
* 支持、犹豫、反对、条件接受的初步分布
* 调研结论生成状态

### 调研结论

顾客全部回复完成后，中间区域展示调研结论。

调研结论至少包含：

* 结论摘要
* 关键发现
* 顾客主要顾虑
* 可接受条件
* 高风险人群
* 建议修改方案
* 下一步行动

### 右侧顾客留声区

右侧与首页“顾客正在想”保持视觉一致，但内容变为本次任务的顾客回答流。

每条顾客留声展示：

* 头像
* 脱敏昵称
* 年龄
* 门店 / 渠道
* 会员身份
* 一段本次任务相关的回答
* 回答状态：等待、正在回答、已回答

不要在右侧展示分析标签，例如“价格敏感型”“支持”“反对”。这些分析结果应沉淀到中间区域的进度和结论里。

### 右侧顾客 hover 快速档案

右侧顾客卡片支持 hover 查看快速档案。

交互要求：

* 鼠标悬浮头像或整张顾客卡片时，出现单层快速档案浮层。
* 同一时刻只允许出现一套弹窗机制，不允许内置浮层和全局浮层同时出现。
* 弹窗应靠近当前顾客卡片，避免遮挡右侧列表的主要内容。
* 头像和卡片需要有轻微 hover 反馈，让用户知道可交互。

快速档案展示：

* 脱敏昵称
* 年龄
* 门店 / 渠道
* 会员身份
* 顾客留言摘要
* 已购商品
* 仿真顾客人设

快速档案不展示：

* 支持 / 反对 / 犹豫 / 条件接受标签
* 价格敏感型等分析标签
* 可编辑输入框

### 信息交互约束

* 首页输入内容必须传递到任务详情页。
* 首页选择的能力按钮应决定任务类型，默认选择“调研问卷”时进入调研问卷流程。
* 首页选择的业务场景应影响任务标题、澄清问题和调研方案。
* 右侧顾客回答必须围绕当前任务生成，不能使用与当前任务无关的固定文案。
* 中间结论必须在顾客回答完成后出现，不应初始展示。

## 页面 3：深度访谈页 / 顾客圆桌模式

### 页面目标

从首页某位仿真顾客的一句话出发，展开一场可追问、可共创、可沉淀结论的深度访谈。

该页面用于验证“仿真顾客不是静态素材，而是可被追问的决策证据来源”。

### 进入方式

用户在首页右侧“顾客正在想”区域点击某位顾客的一句话后进入。

必须传递：

* 顾客脱敏昵称
* 年龄
* 门店 / 渠道
* 会员身份
* 首页展示的一句话
* 已购商品
* 仿真顾客人设

深度访谈页第一条顾客消息必须等于首页点击的那句话，不允许改成固定示例。

### 方案 A：一对一深访

默认进入形态为一对一深访。

布局要求：

* 左侧：收起态平台导航。
* 中间：消息区，借鉴飞书 / 微信客户端的消息阅读节奏。
* 右侧：访谈对象档案，包括脱敏信息、已购商品和仿真顾客人设。
* 底部：追问输入框，可引用资源和知识。

方案 A 不展示：

* 访谈目标卡
* 访谈关键问题卡
* 大面积分析报告
* 支持 / 反对等分析标签

交互要求：

* 顾客首页心声作为第一条消息。
* 用户可以点击“追问原因”或在底部输入框追问。
* 用户追问后展示一条顾客回答。
* 追问后提示用户是否进入“圆桌模式”。

### 方案 C：顾客圆桌模式

当用户确认进入“圆桌模式”后，页面必须切换为独立圆桌工作台。

方案 C 不是在方案 A 上简单增加右侧栏，而是新的信息架构：

1. 顶部：深度访谈任务标题与来源顾客。
2. 参与者区：横向展示圆桌成员。
3. 主体左侧：访谈记录（圆桌对话）。
4. 主体右侧：当前发言者档案和访谈结论草稿。
5. 底部：追问输入区，支持资源、知识和添加参与者。

参与者区至少包含：

* 当前仿真顾客
* 你 / 研究员
* 添加真人
* 添加仿真顾客
* 专家 Agent

右侧“当前发言者档案”至少展示：

* 脱敏昵称
* 年龄
* 门店 / 渠道
* 会员身份
* 关键画像
* 决策红线

右侧“访谈结论草稿”至少展示：

* 顾虑要点
* 接受条件
* 风险信号
* 下一步行动

### 状态切换

当前前端原型使用运行时状态模拟：

* `single`：方案 A，一对一深访。
* `roundtable`：方案 C，顾客圆桌模式。

触发方式：

* 用户追问后点击“是，进入圆桌模式”。
* 用户点击“邀请圆桌”。
* 用户点击添加参与者入口。

验收标准：

* 从首页顾客心声进入深访页后，首条消息与首页心声一致。
* 方案 A 中不出现“访谈目标”和“访谈关键问题”。
* 进入圆桌模式后，页面结构必须明显变成方案 C。
* 圆桌模式中有参与者横排、圆桌对话、右侧档案和结论草稿。

## 页面 4：探索技能详情页

### 页面目标

帮助用户浏览、搜索、启用或停用当前工作台可用的技能。

### 页面结构

参考 AnyGen 的 Skills 页面：

1. 左侧窄图标导航
2. 页面标题和说明
3. 分类胶囊
4. 搜索技能
5. 技能卡片网格

### 技能卡片

技能卡片包含：

* 技能图标
* 技能名称
* 技能说明
* 试用入口
* 使用次数
* 启用状态

悬浮卡片右上角使用开关样式，不使用“+ 添加”。

开关状态：

* 已启用
* 未启用

已启用技能会出现在首页能力按钮和任务推演流程中。

## 页面 5：决策输入页

### 页面目标

帮助企业用户补充一个需要评估的决策问题。

### 页面模块

1. 决策类型选择
2. 决策背景输入
3. 目标顾客选择
4. 当前方案输入
5. 可选方案输入
6. 关键担忧输入
7. 引用资源与知识
8. 开始仿真按钮

### 示例输入

决策标题：

是否将会员年费从 199 元提高到 299 元？

目标顾客：

当前会员用户、潜在会员用户、价格敏感用户、高价值用户。

引用资源：

会员表、价格表、历史续费数据、会员权益说明。

引用知识：

会员体系说明、价格策略原则、历史会员调研。

## 页面 6：仿真顾客群体页

### 页面目标

展示系统为本次决策生成的仿真顾客群体。

### 页面模块

1. 顾客群体总览
2. 仿真顾客卡片
3. 顾客类型分布
4. 顾客关键差异
5. 顾客决策红线
6. 引用资源和知识说明
7. 可调整顾客群体按钮

### 仿真顾客卡片字段

每个仿真顾客包括：

* 姓名或代号
* 顾客类型
* 基本背景
* 核心需求
* 决策方式
* 价格敏感度
* 信任程度
* 主要顾虑
* 决策红线
* 可能行动

## 页面 7：决策推演与反馈页

* 反对

## 页面 8：决策建议报告页

### 页面目标

把仿真反馈转化为企业可执行的决策建议。

### 页面模块

1. 总体决策建议
2. 决策置信度
3. 主要支持理由
4. 主要风险
5. 方案修改建议
6. 顾客沟通建议
7. 小范围测试建议
8. 下一步行动清单
9. 引用资源、知识和推演依据

### 建议类型

最终建议可以分为：

* 建议推进
* 修改后推进
* 先小范围测试
* 暂缓推进
* 不建议推进

### 页面重点

首页是 Agent 工作台，任务内页面是决策驾驶舱。两者要形成连续体验，不能把首页做成普通后台表单，也不能把任务内流程做成闲聊窗口。
