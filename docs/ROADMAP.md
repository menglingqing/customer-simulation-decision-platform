# 路线图

## 阶段 1：可点击 MVP

目标：

做出一个可演示核心产品逻辑的前端原型。

包括：

* 首页 / Agent 工作台
* 决策输入页
* 仿真顾客群体页
* 决策推演与反馈页
* 决策建议报告页
* mock Skill / Tool 数据
* mock 资源库 / 知识库数据
* mock 决策数据
* mock 仿真顾客数据
* mock 顾客反馈数据
* mock 决策报告

重点是验证平台型 Agent 首页、产品逻辑和信息架构。

### 阶段 1 补充：竞品 UI 校准

在继续实现业务页面前，先完成首页和任务详情页的 UI 校准。

参考对象：

* AnyGen：平台型 Agent 首页和任务页布局
* Converge AI：workspace、新建对话 / 新建任务入口、历史记录组织
* ChatGPT：左侧最近记录和低噪音导航
* Lovart：中心输入框和胶囊能力按钮

Converge AI 参考链接：

https://enter.converge.ai/workspace/8fe43b52-efb8-40b8-83a5-59aa0d20560d?tab=newchat

校准原则：

* 不改变产品主线：仿真顾客帮助企业做决策。
* 不取消右侧顾客心声 / 顾客留声区域。
* 不把任务详情页做成普通聊天机器人页面。
* 左侧可以更简洁，但必须保持可读、可发现、可用。
* 首页信息做减法时，优先删除装饰和重复说明，不隐藏核心入口。

## 阶段 2：平台资产体系

目标：

让首页左侧的 Skill 库、Tool 库、资源库、知识库、项目库和历史记录具备基本可浏览结构。

包括：

* Skill 分类与详情
* Tool 分类与详情
* 资源库条目，例如商品表、会员表、仿真顾客表、设计素材
* 知识库条目，例如品牌定位、用户研究、会员体系、历史复盘
* 项目库和最近记录

## 阶段 3：多场景模板

目标：

支持更多企业决策场景。

包括：

* 新产品决策模板
* 定价决策模板
* 营销文案决策模板
* 品牌定位决策模板
* 功能优先级决策模板
* 会员权益决策模板
* 服务政策决策模板
* 渠道策略决策模板

## 阶段 4：仿真顾客系统增强

目标：

提升仿真顾客的稳定性和差异化。

包括：

* 顾客画像配置
* 顾客群体生成规则
* 顾客反馈一致性校验
* 支持多轮追问
* 支持对比不同方案

## 阶段 5：决策推演能力

目标：

让系统能比较多个方案，输出更有价值的决策建议。

包括：

* 多方案对比
* 支持 / 反对分布
* 风险雷达
* 争议点聚类
* 方案优化建议
* 小范围测试建议

## 阶段 6：决策复盘闭环

目标：

让系统记录决策执行后的真实结果，形成企业决策记忆。

包括：

* 决策记录
* 执行结果回填
* 预测与真实结果对比
* 模型和规则修正
* 企业专属顾客理解沉淀
